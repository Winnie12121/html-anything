## 0. 背景与定位

### 0.1 发起背景

博世半导体事业部 HR 团队需要建立一套**竞对公司市场洞察情报体系**，系统性地追踪前十大汽车半导体外资企业的市场动态、人才策略和竞争态势，为团队的人力规划、招聘策略、薪酬对标、人才保留和雇主品牌建设提供数据驱动的决策依据。

### 0.2 报告定位

| 维度      | 定位                                                      |
| --------- | --------------------------------------------------------- |
| **视角**  | HR 视角 — 以人才竞争为核心，产业分析为底座                |
| **深度**  | 从"招多少人"到"招什么人"再到"怎么抢人"                    |
| **频率**  | 月度数据采集 + 季度完整报告，重大事件出临时速报           |
| **产出**  | 完整版（内部HR团队）+ 摘要版（管理层汇报）                |
| **格式**  | **HTML** — 浏览器直接打开，支持打印、响应式布局、交互图表 |
| **执行**  | 自动化采集为主 + AI 辅助分析 + 人工审核定稿               |

## 1. 第一阶段：沟通确认报告框架

### 1.1 目标

### 1.2 关键决策清单

| #   | 决策项        | 选项                                    | 本项目确认                              |
| --- | ------------- | --------------------------------------- | --------------------------------------- |
| 1   | **覆盖范围**  | Top 5 深度 vs Top 10 全覆盖             | ✅ Top 10 全覆盖                        |

## 3. 第三阶段：数据汇总与分析

### 3.1 目标

对采集到的原始数据进行清洗、去重、标准化和交叉验证，然后运用系统化的分析方法，从 HR 视角提炼出可行动的洞察。

### 3.2 数据处理管道

```
raw JSON → 清洗去重 → 标准化 → 结构化写入 Excel → 分析引擎 → 洞察输出
```

| 步骤       | 工具/脚本                                             | 说明                                         |
| ---------- | ----------------------------------------------------- | -------------------------------------------- |
| 清洗去重   | `scripts/parsers/pipeline.py`                         | 按公司+岗位+日期去重，过滤无效/过期数据      |
| JD 解析    | `scripts/parsers/jd_parser.py`                        | 提取技术栈关键词、学历要求、经验年限，打标签 |
| 薪资标准化 | pipeline                                              | 统一 Base/Bonus/RSU 格式，换算年薪总包       |
| 事件分类   | `scripts/crawlers/news_crawler.py` → classify_event() | 投资/并购/建厂/裁员/高管变动/Design Win      |
| 写入 Excel | `scripts/parsers/pipeline.py` → fill\_\*()            | 按 Schema 写入 7 个 Excel                    |

### 3.3 分析方法论全景

>  参考 [数据分析方法论.md](数据分析方法论.md) 中的完整方法体系。

核心分析逻辑：**产业市场分析打底 → 人才数据交叉拆解 → 对标竞品人才策略 → 输出企业人才战略落地建议**。

#### 3.3.1 产业底层分析（理解行业再理解人才）

**a) 产业链价值分层法**

将汽车半导体产业链分为四个层级，每个层级的人才属性、薪酬、供给和竞争完全不同：

| 产业链层级               | 典型岗位                                   | 人才特征                   | HR 用途                             |
| ------------------------ | ------------------------------------------ | -------------------------- | ----------------------------------- |
| **上游** — 材料/设备     | SiC衬底工艺、光刻、靶材、车规设备          | 工艺人才，高校/研究所来源  | 区分高稀缺赛道（SiC器件、功能安全） |
| **中游设计** — 芯片设计  | 车规MCU、功率IC、SoC、模拟、驱动           | 核心设计人才，薪资溢价严重 | 分赛道统计人才存量/流动率/招聘难度  |
| **制造封测** — 生产质量  | 车规晶圆代工、封装、AEC-Q100认证、失效分析 | 工艺/质量人才，稳定性较高  | 制定差异化薪酬带宽                  |
| **下游应用** — Tier1/OEM | 系统集成、软硬件、功能安全                 | 系统级人才，博世优势领域   | 校招专业定向投放策略                |

**b) 赛道生命周期矩阵（四象限）**

```
                    技术壁垒 / 人才门槛
                         ▲
                         │
        高速成长高壁垒    │    新兴萌芽赛道
        (核心紧缺赛道)    │    (复合型稀缺)
        · SiC 功率       │    · 车载存算一体
        · 800V 平台      │    · 车载光芯片
        · 高阶智驾 SoC   │    · 4D 毫米波雷达芯片
        · 功能安全       │
                         │
    ─────────────────────┼─────────────────────→ 市场空间增速
                         │
        成熟赛道          │    红海低壁垒赛道
        · 传统车载 MCU   │    · 普通车载驱动芯片
        · 低压模拟 IC    │    · 低端车身控制芯片
                         │
```

| 象限                            | 人才策略                                   |
| ------------------------------- | ------------------------------------------ |
| **核心紧缺**（高增长×高壁垒）   | 高薪引进 + 股权激励 + 专属保留 + 竞业约束  |
| **成熟赛道**（低增长×低壁垒）   | 内部批量培养 + 标准化晋升 + 适度市场化薪酬 |
| **新兴萌芽**（高增长×不确定）   | 消费电子跨界挖猎 + 产学研联合培养          |
| **红海低壁垒**（低增长×低壁垒） | 严控编制 + 应届生批量填充                  |

**c) 国产替代 SWOT 分析（人才机会与风险）**

| 维度        | 内容                                                                              |
| ----------- | --------------------------------------------------------------------------------- |
| **S 优势**  | 国内整车厂自研、地方产业基金扶持、本土芯片扩产 → 本土人才需求持续上涨             |
| **W 劣势**  | 高端车规成熟人才积累短、资深专家依赖海外/外企回流、功能安全人才缺口大             |
| **O 机会**  | 海外车企半导体裁员、外企技术人才流出、消费电子跨界转车载                          |
| **T 威胁**  | 头部企业（英飞凌/NXP/瑞萨）高薪锁核心专家、扩产扎堆导致人才内卷、短期人力成本暴涨 |

#### 3.3.2 人才数据核心分析（报告量化模块）

**a) 人才供需缺口测算模型（自上而下 + 自下而上交叉校验）**

```
自上而下（需求侧）：
  行业新增企业 + 各企业扩产规划 + 新建研发中心
  → 按人均产出标准测算
  → 3/5年行业总新增岗位需求
  → 拆分赛道：SiC工艺/车规数字设计/功率器件/功能安全/车载软件

自下而上（供给侧）：
  存量：当前从业总人数、分赛道资深专家（5年+车规经验）存量
  增量：微电子/集成电路/材料应届生、可跨界消费电子人才池、海外回流
  流失供给：外企/传统芯片厂流出可流动人才

供需差值 = 行业缺口 → 判断未来2-3年招聘难度
```

**b) 人才价值 × 稀缺度四象限（预算分配依据）**

| 象限               | 人群示例                                 | 人才策略                                   |
| ------------------ | ---------------------------------------- | ------------------------------------------ |
| **高价值+高稀缺**  | SiC器件专家、功能安全经理、智驾SoC架构师 | 高薪引进、股权激励、定制发展通道、竞业约束 |
| **高价值+低稀缺**  | 车载底层软件、普通MCU硬件工程师          | 内部批量培养、标准化晋升、适度市场化薪酬   |
| **低价值+高稀缺**  | 细分小众测试、车规失效分析专家           | 少量核心外部挖猎、其余外包                 |
| **低价值+低稀缺**  | 基础版图、设备操作                       | 严控编制、应届生批量填充                   |

**c) 人才九宫格（评估人才资产健康度）**

```
              绩效 / 项目产出
                    ▲
                    │
    潜力高+产出低    │    潜力高+产出高
    (待激活)         │    (明星人才)
                    │
    ────────────────┼────────────────→ 车载半导体专业潜力
                    │
    潜力低+产出低    │    潜力低+产出高
    (需优化)         │    (骨干力量)
```

横向对标行业数据：
- 行业平均高潜人才占比
- 头部企业明星人才留存率
- 核心技术岗 1/2 年内可继任人才数量

**d) 分层流失归因分析模型**

| 分层维度      | 拆解方向                                           |
| ------------- | -------------------------------------------------- |

| **岗位层级**  | 应届 / 资深工程师 / 经理 / 总监                    |
| **司龄**      | <1年 / 1-3年 / 3-5年 / 5年+                        |
| **企业类型**  | 外企 → 本土上市 / 初创 / 整车自研                  |
| **流失根因**  | 薪酬差距、晋升通道、项目含金量、组织环境、竞品挖角 |

### 3.4 标准分析流程

对每期报告执行以下分析流程：

```
Step 1: PEST + 产业链分层 → 行业发展趋势、细分赛道景气度
Step 2: 赛道生命周期矩阵 → 识别紧缺/饱和技术赛道
Step 3: 人才供需测算 → 量化行业人才缺口
Step 4: 人才价值稀缺四象限 → 分层定义核心战略人才
Step 5: 竞品五维人力对标 → 头部企业人才策略对比
Step 6: 流失 + 薪酬数据分析 → 行业人才流动痛点
Step 7: SWOT 总结 → 企业人才机会与风险
Step 8: 人力规划五步法 → 输出人才战略落地举措
```

将报告框架（第一阶段）+ 采集数据（第二阶段）+ 分析洞察（第三阶段）整合为完整的、可直接使用的市场洞察报告。

```

```

### 4.3 报告生成规则

#### 条件填充模式

每个 Part 采用条件填充逻辑：

```
if Excel 中有相关数据:
    → 渲染数据表格 + 自动化分析文字
else:
    → 显示 "待数据填充" 模板 + 标注需要哪些数据
```

这保证了报告在任何阶段都能输出 — 数据不足时给出明确的数据补充指引，数据充足时自动生成完整内容。
